# AI-developer

**Внутренний ускоритель разработки** для стека Битрикс24 + GitLab. Задача в Битрикс24
проходит брифинг, авторизацию, оценку риска, генерацию кода в изолированной песочнице
**с прогоном проверок репозитория** и превращается в **Draft Merge Request** в GitLab.
Решение о слиянии всегда остаётся за человеком-Maintainer.

Рассчитан на **небольшие механические тикеты** (точечные правки, рутинные изменения).
Крупные изменения система не доводит автоматически — они остаются в Draft для человека.
Доверие держится на **исполняемых проверках** (tests/lint/typecheck, secret-scan);
AI-review и Red Team — блокирующее второе мнение, а не основной источник доверия.

/ Главный принцип: ИИ не догадывается, не мержит и не меняет защищённые контуры; все
рискованные изменения проходят через gates. При любой неопределённости процесс
блокируется и передаётся человеку (**fail closed**). /

## Как читать справочник

Документация построена от общего к частному:

1. **Обзор** — что это, принципы управления рисками, жизненный цикл задачи.
2. **Архитектура** — компоненты, модель данных, конечный автомат брифинга, контракты.
3. **Модули** — справочник по подсистемам и их модулям.
4. **Конфигурация / Безопасность / Тестирование** — сквозные темы.
5. **Развёртывание** — отдельная глава: от локального запуска до production.
6. **Справочник** — команды, эндпоинты, глоссарий.

## Конвейер одним взглядом

```
Битрикс24 задача
  → Intake + валидация
  → Брифинг / уточнения → /go (с авторизацией)
  → Risk scoring → Plan Gate
  → Explore & Plan (песочница)
  → Кодинг с прогоном проверок репо до зелёного (verified)
  → Draft MR (ветка auto-task-* от main → MR в dev)
  → Бюджет scope: малый дифф → авто Ready, крупный → остаётся Draft человеку
  → CI/CD → AI-review → Red Team (если нужно)
  → Human review → Merge by Maintainer
```
