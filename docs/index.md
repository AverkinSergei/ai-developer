# AI-developer

Автономная AI-система разработки: задача в Битрикс24 проходит брифинг, авторизацию,
оценку риска, генерацию кода в изолированной песочнице и превращается в **Draft Merge
Request** в GitLab. Решение о слиянии всегда остаётся за человеком-Maintainer.

/ Главный принцип: ИИ не догадывается, не мержит и не меняет защищённые контуры; все
рискованные изменения проходят через gates. При любой неопределённости процесс
блокируется и передаётся человеку (**fail closed**). /

## Как читать справочник

Документация построена от общего к частному:

1. **Обзор** — что это, принципы управления рисками, жизненный цикл задачи.
2. **Архитектура** — компоненты, модель данных, конечный автомат брифинга, контракты.
3. **Модули** — справочник по подсистемам и их модулям.
4. **Конфигурация / Безопасность / Тестирование** — сквозные темы.
5. **Развёртывание** — отдельная глава: от локального запуска до production.
6. **Справочник** — команды, эндпоинты, глоссарий.

## Конвейер одним взглядом

```
Битрикс24 задача
  → Intake + валидация
  → Брифинг / уточнения → /go (с авторизацией)
  → Risk scoring → Plan Gate
  → Explore & Plan (песочница)
  → Кодинг + Draft MR (ветка auto-task-* от main → MR в dev)
  → CI/CD → AI-review → Red Team (если нужно)
  → Human review → Merge by Maintainer
```
