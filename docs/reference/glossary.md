# Глоссарий

| Термин | Значение |
|---|---|
| **Брифинг** | этап уточнения постановки вопросами-раундами до выполнения DoR |
| **DoR** (Definition of Ready) | условия, при которых задача допускается к Explore & Plan |
| **DoD** (Definition of Done) | условия готовности MR к приёмке человеком |
| **`/go`** | управляющая команда: закрыть брифинг и разрешить работу (с авторизацией) |
| **Plan Gate** | гейт между планом и кодингом; для high-risk — pre-approval человека |
| **Risk scoring** | классификация задачи `low/medium/high/blocked` |
| **Red Team** | независимый security-контур поиска эксплойтов |
| **AI-review** | независимый ревьюер диффа, изолированный от Coder |
| **Песочница** | безопасные `list_dir`/`grep`/`read_file` по корню checkout |
| **Self-fix** | петля автоправок по CI/review в пределах `MAX_AI_FIXES` |
| **verified / unverified** | MR с прогнанными зелёными проверками репо / без них (unverified смотрит человек) |
| **Бюджет scope** | лимит фактического диффа (строк/файлов); превышение оставляет MR в Draft человеку |
| **Outbox** | транзакционный журнал намерений enqueue (защита от lost-start) |
| **`auto-task-{id}`** | ветка агента; форкается от `main`, MR нацелен в `dev` |
| **fail closed** | при неопределённости — блокировка и передача человеку |
| **Maintainer** | человек, выполняющий merge; единственный, кто мержит |
| **Controlled AI contributor** | роль системы: готовит изменения, но не выпускает в prod |
