# Команды

Команды пишутся в комментариях задачи Битрикс24 (брифинг, `/go`) или в комментариях MR
GitLab (`@ai *`). Распознаются только в начале комментария.

## Брифинг и старт

| Команда | Кто может | Значение |
|---|---|---|
| `/go` | постановщик/ответственный/reviewer/maintainer/allowlist | закрыть брифинг и разрешить работу (после авторизации) |
| `/briefing answer <round_id>` | авторизованный участник | ответы на вопросы раунда |
| `/briefing status` | участник | показать состояние брифинга |
| `/briefing reopen <reason>` | участник | новый раунд после READY_FOR_GO |
| `/briefing cancel <reason>` | участник | остановить брифинг |
| `/briefing skip <question_id> <reason>` | участник | отказать в ответе на вопрос |

## Команды по ходу работы (`@ai`)

| Команда | Кто может | Значение |
|---|---|---|
| `@ai status` | любой участник | краткий статус фазы и блокеров |
| `@ai stop` | постановщик/reviewer/maintainer | остановить агента и снять lock |
| `@ai retry` | постановщик/reviewer | перезапустить упавшую фазу |
| `@ai resolve` | reviewer/maintainer | разрешить конфликт MR |
| `@ai fix` | reviewer/maintainer | исправить замечания в MR в рамках лимитов |
| `@ai redteam` | reviewer/maintainer/security | принудительно запустить Red Team |

## CLI

| Команда | Назначение |
|---|---|
| `python -m app.cli build-graph <repo...> [--ref]` | построить графы кода репозиториев в `GRAPH_CACHE_DIR` (алиас `ai-developer build-graph`, если пакет установлен) |

## Пример раунда

```
[AI_BRIEFING] session_id: brf_B24-1  round_id: r1  status: WAITING_ANSWERS
Ответьте командой: /briefing answer r1
Q1: Какой критерий успешной приёмки?
Q2: Какие роли должны иметь доступ?
```
Ответ постановщика:
```
/briefing answer r1
A1: пользователь видит статус заказа без перезагрузки
A2: доступ только у менеджера и администратора
```
