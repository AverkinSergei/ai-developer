# Введение

## Назначение

Система автоматизирует подготовку изменений по задачам разработки: анализ постановки,
брифинг, планирование, генерацию кода и документации, создание ветки и Draft MR, прогон
проверок репозитория, AI-review, Red Team-review при необходимости и ограниченный цикл
самоисправления.

Это **внутренний ускоритель** для своего стека Битрикс24 + GitLab, а не универсальный
продукт и не автономный разработчик с правом выпуска в production. Точнее — *controlled
AI contributor*: он создаёт изменения в изолированной ветке, но не выполняет merge, не
управляет protected branches и не получает доступ к production-секретам.

## Позиционирование и границы scope

Целевой класс задач — **небольшие механические тикеты**: точечные правки, рутинные
изменения, шаблонная работа. Объём автономной работы ограничен бюджетом scope (число
изменённых файлов и строк диффа): малый дифф система доводит до состояния Ready, крупный
оставляет в Draft и передаёт человеку. Так агент берёт на себя рутину, а нетривиальные
изменения остаются под контролем разработчика.

Доверие к изменению держится на **исполняемых проверках** репозитория (tests / lint /
typecheck, secret-scan), которые гоняются в песочнице до зелёного. AI-review и Red Team —
блокирующее **второе мнение**, а не основной источник доверия: они могут заблокировать
Ready, но «зелёным» MR делают именно прогнанные проверки.

## Зрелость

Сквозная бизнес-логика (intake → брифинг → `/go` → план → код с проверками → Draft MR →
гейты) реализована и покрыта тестами на fake-клиентах. **Живой прогон** против реальных
GitLab / Битрикс24 / LLM ещё не проводился — это следующий шаг (пилот на 10–20 реальных
тикетах). Разрешение конфликтов MR (`@ai resolve`) пока ограничено: полноценное 3-way
слияние не делается, конфликт маршрутизируется человеку с диагностикой.

## Покрываемые сценарии

| Тип задачи | Контур | Результат |
|---|---|---|
| `feature` | кодинг → GitLab | многофайловые изменения + Draft MR |
| `bugfix` | кодинг → GitLab | точечная правка + Draft MR |
| `refactor` | кодинг → GitLab | изменения без новой логики + Draft MR |
| `research` | исследование → Б24 | markdown-отчёт в задаче, без ветки/MR |
| `review` | комментарии в MR | без push, если не запрошен `@ai fix` |
| `security_review` | Red Team | findings + рекомендации |

## Что система не делает автономно

Срочные production-инциденты, ручные миграции боевых данных, изменения IAM/секретов,
отключение проверок CI/CD, изменение настроек protected branch. Такие задачи
останавливаются и передаются человеку.
